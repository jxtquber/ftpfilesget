# 鸿蒙 FTPS 客户端需求文档

## 1. 产品目标

开发一个鸿蒙 ArkTS 应用，用于连接显式 FTPS 服务器、浏览远程目录，并把远程文件下载到用户自己选择的本地位置。

应用定位是一个直接可用的工具，不做营销页。打开应用后的第一个页面就是连接表单，连接成功后进入远程文件列表页面。

## 2. 目标平台

- 平台：HarmonyOS
- UI 技术：ArkTS / ArkUI
- 工程类型：带 `entry` 模块的鸿蒙应用
- 必需权限：
  - `ohos.permission.INTERNET`
- 下载文件时不要申请宽泛的文件读写权限。应使用系统文档保存选择器，让用户选择保存位置，由系统授予该文件 URI 的写入权限。

## 3. 默认连接参数

连接页面默认值：

- 服务器地址：`192.168.1.101`
- 服务器地址下拉选项：
  - `192.168.1.101`
  - `juneinter.site`
- 服务器地址仍然必须支持手动输入。
- 端口：`21`
- 用户名：`jxtquber`
- 密码：默认为空，用户每次进入应用都需要手动输入。

需要本地持久化保存：

- 服务器地址
- 端口
- 用户名

不要保存密码。

## 4. 应用图标

应用图标必须能清楚表达 FTP / 文件传输含义。

必需视觉元素：

- 文件夹或文件图形
- 双向箭头或传输箭头
- 明确文字：`FTP`

以下资源使用同一套图标概念：

- `AppScope/resources/base/media/app_icon.png`
- `entry/src/main/resources/base/media/icon.png`

图标保持正方形，兼容鸿蒙启动器资源要求。当前实现使用 `108x108` PNG 资源。

## 5. 页面结构

### 5.1 连接页面

用途：填写连接信息并进入文件浏览器。

布局要求：

- 页面整体居中。
- 背景使用浅色中性色。
- 表单使用白色卡片，带轻微边框和阴影。
- 标题：`FTPS文件浏览器`
- 副标题：`连接并浏览远程文件`
- 卡片内输入行：
  - 左侧标签，右侧输入框或控件
  - 标签包括：
    - `IP地址`
    - `端口号`
    - `用户名`
    - `密码`
- IP 地址行：
  - 使用可编辑输入框。
  - 输入框右侧带紧凑的下拉箭头图标。
  - 展开后显示两个可选项：
    - `192.168.1.101`
    - `juneinter.site`
  - 不要使用可见的 `选择` / `收起` 文字按钮。
- 密码行：
  - 使用密码输入框。
  - 不显示密码可见性切换按钮。
- 主按钮：
  - 文案：`连接服务器`
  - 正在连接、服务器地址为空、密码为空时禁用。
- 页面只显示简洁的连接错误。
- 最终 UI 不显示诊断信息面板。
- 最终 UI 不显示协议说明文字。

退出行为：

- 在连接页面按系统返回键时，弹出居中的退出确认弹窗。
- 弹窗必须有清晰的卡片或边框。
- 按钮：
  - `取消`
  - `退出`

### 5.2 文件列表页面

用途：浏览远程目录并下载文件。

顶部栏：

- 左侧按钮：`返回`
- 中间显示当前路径，路径过长时省略。
- 右侧按钮：`刷新`

返回行为：

- 当前路径为 `/` 时，点击 `返回` 弹出退出确认弹窗。
- 当前路径不是 `/` 时，点击 `返回` 加载父目录。
- 文件列表中不要再显示单独的 `..` 返回行，避免和左上角返回按钮冲突。
- 下载过程中，系统返回键和左上角 `返回` 按钮都不可用。

刷新行为：

- 点击 `刷新` 时必须关闭旧的 FTPS 控制连接，创建新的 FTPS 客户端，重新登录，并加载当前路径。
- 加载中或下载中时禁用 `刷新`。

目录进入行为：

- 点击文件夹时，用当前长连接打开对应目录。
- 如果控制连接已被服务器断开，应自动重连并重试一次。
- 目录加载过程中：
  - 禁用继续点击文件夹、刷新和下载按钮。
  - 显示转圈加载状态。
  - 如果旧文件列表仍然可见，则列表变暗并显示居中的加载遮罩。
  - 条件允许时，在正在打开的文件夹行上显示小转圈。

文件列表行布局：

- 所有文件名、文件夹名左对齐。
- 每一行最左侧显示类型图标。
- 文件夹和不同文件类型使用不同图标或颜色。
- 文件行显示：
  - 图标
  - 名称
  - 修改时间
  - 文件大小
  - 下载按钮
- 文件夹行显示：
  - 图标
  - 名称
  - 修改时间
  - 不显示下载按钮

文件类型图标和颜色示例：

- 文件夹：金色文件夹样式图标
- 图片：绿色图标或下载按钮
- 视频：紫色图标或下载按钮
- 音频：橙色图标或下载按钮
- 压缩包：灰色图标或下载按钮
- 文档：蓝色图标或下载按钮
- 代码文件：红色图标或下载按钮
- 其他类型：中性颜色图标或下载按钮

下载行为：

- 点击 `下载` 后，首先打开系统文档保存选择器。
- 选择器默认文件名使用远程文件名，并过滤本地非法字符。
- 如果用户取消选择：
  - 不启动 FTPS 下载。
  - 可以显示简短的取消提示。
- 如果用户选择了保存位置：
  - 启动 FTPS 下载。
  - 底部显示下载进度区域：
    - `正在下载: <文件名>`
    - 进度条
  - 下载过程中禁用返回、刷新、目录进入和其他下载按钮。
- 下载成功：
  - 弹出 Toast：`下载完成`
  - 不要在顶部用红色文字显示成功信息。
- 下载失败：
  - 顶部显示错误：`下载失败: <具体错误>`
  - 传输失败后要重连 FTPS 控制连接，避免旧响应污染下一次 `PASV` / `EPSV` 解析。

退出弹窗：

- 居中显示。
- 有明确卡片或边框。
- 文案：
  - 标题：`确认退出`
  - 内容：`确定要退出文件浏览器吗？`
- 按钮：
  - `取消`
  - `确定退出`
- 确认退出时：
  - 断开 FTPS 连接
  - 必要时清理路由
  - 退出应用

## 6. FTPS 协议要求

### 6.1 连接模式

需要支持：

- 端口 `21` 的显式 FTPS
- 端口 `990` 的隐式 FTPS

协议判断：

- 如果端口是 `990`，从一开始就建立 TLS 连接。
- 其他端口先建立普通 TCP 连接，再通过 `AUTH TLS` 升级到 TLS。

### 6.2 显式 FTPS 登录流程

显式 FTPS 流程：

1. 打开到服务器的 TCP socket。
2. 读取服务器欢迎响应，期望状态码为 `220`。
3. 发送 `AUTH TLS`。
4. 如果 `AUTH TLS` 失败，可以尝试 `AUTH SSL`。
5. 期望服务器返回 `234`。
6. 将已有 TCP socket 包装为 TLS socket。
7. TLS 协议使用 `TLSv12`。
8. 为兼容局域网或自签名证书服务器，跳过远端证书校验。
9. 发送：
   - `PBSZ 0`
   - `PROT P`
10. 发送：
   - `USER <用户名>`
   - `PASS <密码>`
11. 登录成功后发送：
   - `TYPE I`

### 6.3 隐式 FTPS 登录流程

隐式 FTPS 流程：

1. 直接打开 TLS socket。
2. 读取服务器欢迎响应，期望状态码为 `220`。
3. 发送：
   - `PBSZ 0`
   - `PROT P`
4. 使用 `USER` / `PASS` 登录。
5. 发送 `TYPE I`。

### 6.4 控制连接

- 文件列表页面生命周期内保持 FTPS 控制连接不断开。
- 目录导航和文件下载复用同一个控制连接。
- 只有以下情况才断开：
  - 用户退出文件列表页面或退出应用
  - 用户点击刷新
  - 传输失败后需要重连
  - 检测到连接已关闭，需要重建连接

### 6.5 数据连接

FTP / FTPS 的 `LIST` 和 `RETR` 都需要独立数据连接。

每次列目录或下载文件时：

1. 打开被动模式数据端点：
   - 优先使用 `EPSV`
   - 如果 `EPSV` 没有返回 `229`，再使用 `PASV`
2. 解析端点：
   - EPSV 格式：`(|||端口|)`
   - PASV 格式：`(h1,h2,h3,h4,p1,p2)`
3. 打开数据 TCP socket。
4. 发送控制命令：
   - 列目录：`LIST <路径>`
   - 下载：`RETR <路径>`
5. 期望服务器返回 `1xx` 预备响应。
6. 将数据 TCP socket 包装为 TLS socket。
7. 读取数据。
8. 等待控制连接返回最终 `2xx` 完成响应。
9. 关闭数据连接。

### 6.6 目录列表读取优化

局域网内进入目录应尽量快。

实现要求：

- 读取文本列表数据时，如果已经收到数据，不要无限等待服务器关闭数据连接。
- 可以使用短空闲计时器，例如最后一个数据块到达后等待 150 ms 即认为列表数据读取完成。
- 仍然需要读取控制连接上的最终完成响应。

### 6.7 下载读取行为

- 二进制下载需要持续收集所有数据块，直到数据连接关闭或出错。
- 如果下载超时或失败：
  - 关闭数据连接。
  - 在允许下一次传输前，重连控制连接。

## 7. 状态管理

关键 UI 状态：

- `isLoading`
- `loadingPath`
- `downloadingFile`
- `downloadProgress`
- `currentPath`
- `errorMessage`

状态规则：

- `isLoading` 会阻止刷新、目录点击和下载。
- `downloadingFile` 会阻止返回、刷新、目录点击和其他下载。
- `loadingPath` 用于标识当前正在打开的目录。
- `errorMessage` 只显示错误或有意义的取消信息，不显示下载成功。

## 8. 错误处理

连接页面：

- 显示简洁连接错误：
  - `请输入服务器地址`
  - `请输入密码`
  - `连接失败: <错误信息>`

文件列表页面：

- 加载目录：
  - 第一次失败后自动重连并重试一次。
  - 重试仍失败时显示：`加载失败: <错误信息>`。
- 刷新：
  - 重连或重新加载失败时显示：`刷新失败: <错误信息>`。
- 下载：
  - 用户取消选择保存位置时，可以显示 `已取消下载`，也可以不显示。
  - 成功时只弹 Toast：`下载完成`。
  - 失败时显示：`下载失败: <错误信息>`。
  - 传输失败后必须重连控制连接。

避免旧响应污染：

- UI 层阻止并发 FTPS 操作。
- 操作进行中禁用重复刷新、下载和目录点击。
- 一次传输超时后，下一次操作前必须重连。

## 9. 文件名处理

把远程文件名传给保存选择器前，需要处理本地非法文件名字符：

- 替换以下字符：
  - `\ / : * ? " < > |`
- 去掉首尾空白。
- 如果处理后为空，使用 `download_file`。

## 10. 本地持久化

使用 preferences 保存：

- `address`
- `port`
- `username`

不要保存：

- 密码

## 12. 验收标准

连接：

- 应用打开后显示居中的连接表单。
- IP 默认值为 `192.168.1.101`。
- IP 下拉提供 `192.168.1.101` 和 `juneinter.site`。
- IP 输入框仍然可以手动编辑。
- IP 下拉不使用可见的 `选择` / `收起` 文字按钮。
- 端口默认值为 `21`。
- 用户名默认值为 `jxtquber`。
- 每次启动应用时密码为空。
- 不保存密码。
- 连接页面不显示密码可见性切换按钮。
- 不显示诊断信息面板。

浏览：

- 可以成功连接端口 21 的显式 FTPS。
- 可以显示根目录文件列表。
- 文件夹行和文件行都左对齐。
- 不同文件类型有不同图标和下载按钮颜色。
- 左上角返回按钮可以返回父目录。
- 文件列表中不显示 `..` 行。
- 刷新会重连并重新加载当前目录。
- 文件夹导航复用长控制连接。
- 加载状态能防止重复点击。

下载：

- 点击下载后打开系统保存选择器。
- 使用用户选择的保存位置。
- 下载时显示进度。
- 下载中禁止返回。
- 下载成功只显示 Toast。
- 下载失败显示清晰错误。
- 一次超时或失败后，下一次下载不会因为旧响应导致 `PASV解析失败`。

退出：

- 退出确认弹窗居中且有边框或卡片。
- 退出文件列表或应用时断开 FTPS。

图标：

- 启动器图标能清楚表达 FTP / 文件传输。

## 13. 从零重建说明

如果代码丢失，需要让 AI 编程助手根据本文档重建应用，可以直接把本文档作为完整需求来源，并要求它：

1. 创建鸿蒙 ArkTS 应用工程。
2. 实现上面列出的四个主要文件。
3. 只申请 `INTERNET` 权限。
4. 下载使用 `DocumentViewPicker.save()`。
5. 完整实现 FTPS，包括 `AUTH TLS`、`PBSZ`、`PROT`、`EPSV` / `PASV`、`LIST`、`RETR` 和 TLS 数据连接。
6. 确保 ArkTS 代码不使用 `any` 和 `unknown`。
7. 替换启动器资源，使用明确的 FTP 图标。

不要在源码中硬编码密码。
